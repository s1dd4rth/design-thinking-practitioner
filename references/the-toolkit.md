# The Toolkit — guided activities to run in a session

These are the named IBM Enterprise Design Thinking activities. The Loop tells you which *mode* the team is in; this catalog tells you which *activity* to actually run. Pick one, then facilitate it live — ask its prompts one at a time and fill its structure with the team's answers.

Each entry: what it produces and the core prompts you drive. Activities are grouped the way IBM groups them; the **Loop mode** tag tells you when each fits.

---

## Gather user data — (Observe)
**Interviews** — get to know users/stakeholders by talking to them. *Facilitate:* draft open-ended, non-leading questions about how they live and work; sequence warm-up → story → needs → wrap. Output: an interview guide.
**Contextual Inquiry** — observe users in their daily context to understand the *how* and *why* behind behavior. *Facilitate:* plan what to watch, where, and what not to interrupt. Output: an observation plan + capture sheet.

## Synthesize research — (Reflect, some Observe)
**Empathy Map** — build empathy from the team's observations. *Facilitate:* fill **Says / Thinks / Does / Feels** for one user; then ask "what surprises us? what contradicts?" Output: a populated empathy map.
**As-Is Scenario Map** — understand the user's *current* experience. *Facilitate:* lay out phases across the top; for each, capture what the user is **doing / thinking / feeling**. Mark the lowest emotional points as opportunities. Output: an as-is map with pain points.
**Needs Statements** — outline what users need to reach their goals. *Facilitate:* phrase each as *[User] needs a way to [goal] because [insight].* Keep them solution-free. Output: a prioritized list of needs.
**Stakeholder Map** — identify stakeholders, expectations, and relationships. *Facilitate:* place the user at center; map who influences/depends on whom. Output: a stakeholder diagram.

## Plan your work — (Reflect)
**Assumptions and Questions** — a reality check. *Facilitate:* split what the team *assumes* from what it *knows*; convert risky assumptions into questions to test. Output: an assumptions/questions table with test plan.
**Hills** — state intent as user outcomes (Who/What/Wow). *See `the-keys.md` for full facilitation.*
**Prioritization Grid** — decide the next move on **importance × feasibility**. *Facilitate:* plot items on the 2×2; act on high-importance/high-feasibility first. Output: a ranked shortlist.
**Experience-based Roadmap** — break a long-term experience into essential near-term outcomes. *Facilitate:* sequence outcomes by user value over now/next/later. Output: an outcome roadmap.
**Research Plan** — define research goals and steps against the team's unknowns. *Facilitate:* list unknowns → method per unknown → who/when. Output: a research plan.
**User Stories** — requirements as discrete user tasks. *Facilitate:* *As a [user], I want to [task], so that [outcome].* Output: a story list (pairs well after Playback Zero).

## Generate new ideas — (Make)
**Big Idea Vignettes** — rapidly diverge on many possible solutions. *Facilitate:* time-boxed quantity-over-quality generation; defer judgment; then cluster. Output: a breadth of captioned ideas.
**To-be Scenario Map** — vision of the user's *future* experience. *Facilitate:* re-walk the As-Is phases showing how ideas change doing/thinking/feeling. Output: a to-be map.
**Storyboards** — communicate an idea as a visual sequence in the user's life. *Facilitate:* 4–8 frames: setup → problem → solution moment → outcome. Output: a storyboard.
**Speculative Design Stories** — short stories predicting users' *future* problems and novel solutions. *Facilitate:* pick a trend, extrapolate, write the user's future day. Output: a speculative narrative.

## Test your ideas — (Make + Observe)
**Paper Prototypes** — cheap, fast, versatile idea communication. *Facilitate:* decide the single question the prototype must answer; build only enough to ask it. Output: a paper prototype plan.
**Wireframes** — low/mid-fidelity UI to get quick feedback. *Facilitate:* pick fidelity by feedback needed; one screen per key decision. Output: a wireframe spec.
**Cognitive Walkthrough** — observe users interacting with a prototype to evaluate success. *Facilitate:* define the task; at each step ask "will the user know what to do, and that they did it?" Output: a walkthrough script + findings sheet.
**Feedback Grid** — organize feedback from users/team/stakeholders. *Facilitate:* sort into **Works / Doesn't / Questions / Ideas** (4 quadrants). Output: a populated feedback grid.

## Align your team — (Reflect)
**Hopes and Fears** — surface each person's hopes and fears at kickoff or onboarding. *Facilitate:* round-robin capture; cluster; name the top shared fear to address. Output: a hopes/fears board.
**Daily Syncs** — stay informed, ask for help, surface blockers early. *Facilitate:* each member: doing / blocked-on / need. Output: a sync format.
**Playbacks** — tell stories to share work and exchange feedback. *See `the-keys.md`.*
**Retrospectives** — reflect at an iteration's end to improve how the team works. *Facilitate:* what to keep / stop / start; pick one experiment. Output: a retro summary + one action.

## Define your AI strategy — (cross-cutting, for AI-enabled problems)
A five-part sequence to decide *whether and how* AI should solve a user's problem:
1. **Intent** — determine why you would or wouldn't use AI here. (Start here — don't default to AI.)
2. **Data** — identify the data you have vs. the data you need.
3. **Understanding** — deconstruct data sources into what's needed to teach the AI.
4. **Reasoning** — revisit big ideas and ground them in feasibility.
5. **Knowledge** — brainstorm the AI's impact on people and the world.
*Facilitate:* run in order; Intent gates the rest. Output: an AI strategy brief.

## Picking the right tool
- Don't know the user → **Interviews / Contextual Inquiry**, then **Empathy Map / As-Is Map**.
- Drowning in notes → **Affinity** (see the-loop) → **Needs Statements** → **Prioritization Grid**.
- Need scope → **Hills**, then **Experience-based Roadmap**.
- Need ideas → **Big Idea Vignettes** → **To-be Map / Storyboards**.
- Need to test → **Paper Prototypes / Wireframes** → **Cognitive Walkthrough** → **Feedback Grid**.
- Team friction / kickoff → **Hopes and Fears**, **Daily Syncs**, **Retrospectives**.
- It's an AI problem → run **AI Essentials** (Intent first).

Always close by naming the artifact produced and the next activity in the loop.
